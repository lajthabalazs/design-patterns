# Design Patterns

A collection of design pattern implementations in Java.

## Creational Patterns

Creational patterns manage the instantiation of objects.

| Pattern | Description |
|---------|-------------|
| [Abstract Factory](src/main/java/com/lajthabalazs/designpatterns/creation/abstractfactory/README.md) | Create families of related objects without specifying their concrete classes |
| [Factory Method](src/main/java/com/lajthabalazs/designpatterns/creation/factorymethod/README.md) | Define an interface for creating an object, letting subclasses decide which class to instantiate |

## Structural Patterns

Structural patterns organize objects into larger functional structures.

| Pattern | Description |
|---------|-------------|
| [Adapter](src/main/java/com/lajthabalazs/designpatterns/structure/adapter/README.md) | Convert the interface of a class into another interface that clients expect |
| [Decorator](src/main/java/com/lajthabalazs/designpatterns/structure/decorator/README.md) | Enhance the functionality of an object at runtime using composition |
| [Flyweight](src/main/java/com/lajthabalazs/designpatterns/structure/flyweight/README.md) | Reduce memory usage by sharing common state across many objects |

## Behavioral Patterns

Behavioral patterns manage communication and responsibility between objects.

| Pattern | Description |
|---------|-------------|
| [Chain of Responsibility](src/main/java/com/lajthabalazs/designpatterns/behavior/chainofresponsibility/README.md) | Pass requests along a chain of handlers, each deciding to handle or forward |
| [Strategy](src/main/java/com/lajthabalazs/designpatterns/behavior/strategy/README.md) | Define a family of interchangeable algorithms behind a common interface |
| [Visitor](src/main/java/com/lajthabalazs/designpatterns/behavior/visitor/README.md) | Execute operations on every element of a data structure without modifying the elements |
